# QuantDash AI MCP Guide

## 目标

这份文档给后续接入的 AI 使用。

AI 不应该解析前端页面 DOM，也不应该依赖截图来判断盘面。
AI 应该直接通过本项目提供的 MCP Server 读取结构化数据，再做分析。

## MCP Server

- 启动命令: `npm run mcp:server`
- 入口文件: `scripts/mcp-server.js`
- 运行方式: `stdio MCP server`

如果你要接飞书问答机器人：

- 启动命令: `npm run feishu:bot`
- 入口文件: `scripts/integrations/feishu/bot-server.js`
- 事件接收方式: `长连接 / WebSocket`
- 不需要公网回调 URL
- 飞书服务和 `get_sentiment_snapshot` 使用同一份本地情绪快照构建逻辑

推荐在 MCP 客户端中这样注册:

```json
{
  "mcpServers": {
    "quantdash": {
      "command": "npm",
      "args": ["run", "mcp:server"],
      "cwd": "C:\\Users\\lenvon\\Desktop\\quantdash---a-share-analytics"
    }
  }
}
```

## 当前可用 Tools

### 1. `get_market_dashboard`

用途:
返回当前盘面的总览快照，适合 AI 先做全局判断。

返回内容重点:

- 最新交易日
- 涨停结构
- 修复率
- 龙头状态
- 题材持续性
- 新闻摘要

推荐用途:

- 判断市场处于试错、主升、分歧、退潮还是修复
- 生成开盘前/收盘后盘面摘要
- 作为 AI 分析的第一步输入

### 2. `get_leader_state`

用途:
返回最近几天的核心龙头状态历史。

输入参数:

- `days`: 3 到 10，默认 5

返回内容重点:

- 每日核心龙头
- 最高板
- 同高度龙头数
- 3板以上家数
- 次日开盘反馈
- 次日收盘反馈
- 状态标签

状态标签说明:

- `one_word_acceleration`: 一字加速
- `strong_continuation`: 强势晋级
- `accepted_after_break`: 断板承接
- `high_level_divergence`: 高位分歧
- `weakening`: 分歧转弱
- `retreat_pressure`: 退潮承压
- `await_confirmation`: 待次日确认
- `high_leader_cluster`: 高标抱团
- `one_word_watch`: 一字观察

推荐用途:

- 判断高标是否继续可做
- 判断市场是否由主升转分歧
- 判断是否进入高位退潮

### 3. `get_sector_persistence`

用途:
返回题材持续性数据。

输入参数:

- `boardType`: `concept` 或 `industry`

返回内容重点:

- 当前最强题材
- 连续领涨天数
- 近 5 日 Top3 出现次数
- 近 5 日最常出现的强题材
- 每日题材持续性轨迹

推荐用途:

- 判断主线是否持续
- 判断题材是日内脉冲还是阶段主线
- 辅助判断做主线还是做轮动补涨

### 4. `get_news_feed`

用途:
返回本地新闻流。

输入参数:

- `limit`: 1 到 100，默认 20
- `source`: 默认 `all`

返回内容重点:

- 新闻标题
- 时间
- 来源
- 情绪标签
- 类型
- 原文链接

推荐用途:

- 结合盘面强弱解释异动原因
- 做盘中资讯摘要
- 给 AI 提供消息面背景

### 5. `get_cycle_overview`

用途:
返回当前市场阶段、量能状态、风险等级和核心判断依据。

返回内容重点:

- 当前阶段
- 置信度
- 风险等级
- 量能状态
- 3 条核心依据

推荐用途:

- 作为 AI 最先读取的结论层输入
- 快速判断今天该防守还是进攻

### 6. `get_volume_trend`

用途:
返回近几日量能趋势和量能状态。

返回内容重点:

- 最新成交额
- 日度变化率
- 量能状态
- 历史趋势序列

推荐用途:

- 判断持续放量、存量震荡、缩量再缩量、放量滞涨
- 辅助识别周期是否具备持续性

### 7. `get_high_risk_panel`

用途:
返回高位风险面板数据。

返回内容重点:

- 4板以上样本数
- 高位A杀数量
- 高位转弱数量
- 炸板率
- 风险等级

推荐用途:

- 判断高位亏钱效应是否扩散
- 辅助识别退潮是否开始

### 8. `get_sentiment_snapshot`

用途:
返回情绪周期页面的核心结构化快照，支持指定某一个交易日。

输入参数:

- `date`: 可选，格式 `YYYY-MM-DD`。不传时默认返回最新可用交易日。

返回内容重点:

- 砸盘系数
- 涨停溢价 / 次日成功率 / 炸板率
- 涨停结构
- 修复率
- 龙头状态
- 高度趋势
- 牛熊风向标
- 情绪指标
- 股指期货多空数据

推荐用途:

- 让 AI 单独分析某一天盘面
- 对比昨天和今天的情绪差异
- 让 AI 综合参考砸盘系数、涨停溢价、修复率和龙头状态做盘后判断

### 9. `get_research_reports`

用途:
列出本地研报目录里的文件清单。

返回内容重点:

- 文件名
- 相对路径
- 绝对文件路径
- 扩展名
- 文件大小
- 更新时间
- 预览类型
- 原始 URL
- 建议读取方式

推荐用途:

- 让 AI 先选择要读哪份研报
- 给 AI 展示当前本地有哪些资料可用
- 给支持 PDF/图片/Office 的 AI 直接提供文件定位信息

### 10. `get_research_report_content`

用途:
读取单份研报内容。

输入参数:

- `reportId`: 由 `get_research_reports` 返回

说明:

- 对文本类文件会直接返回内容
- 对 PDF、图片、Office 等非文本文件，会返回元数据和提示信息
- 返回值会带文件绝对路径，方便支持文件输入的 AI 直接读取原文件

推荐用途:

- 让 AI 读取手动放入的文本型研报
- 结合盘面数据做研报解读

### 11. `get_expert_holding_snapshots`

用途:
列出本地同步好的草原高手 CSV 快照。

返回内容重点:

- 快照日期
- 原始文件名
- 原始 CSV 路径
- 记录数
- 覆盖组别
- 当日正收益人数
- 空仓人数
- 建议读取方式

推荐用途:

- 让 AI 先判断要读取哪一天的高手样本
- 快速识别当天是激进持仓还是偏防守

### 12. `get_expert_holding_snapshot`

用途:
读取单日草原高手持仓明细。

输入参数:

- `snapshotId`: 快照日期或 id，例如 `2025-12-15`

返回内容重点:

- 单日完整持仓记录
- 正收益人数
- 负收益人数
- 空仓人数
- 当日收益前五
- 当日收益后五
- 原始 CSV 路径

推荐用途:

- 让 AI 分析高手群体是否集中进攻某个方向
- 辅助判断情绪是否偏进攻、分歧还是偏防守

## 数据口径

### 龙头状态

口径:

- 以每日涨停池中的最高连板股作为核心龙头样本
- 如果同高度有多只，优先取更早封板者作为代表
- 再计算次日反馈和状态标签

### 修复率

口径:

- `brokenRepairRate`: 昨日炸板股中，次日收盘红盘的占比
- `bigFaceRepairRate`: 昨日炸板池里跌幅大于等于 5% 的个股中，次日收盘红盘的占比

说明:

- 目前是实时计算
- 当前 MCP 里的总览会带修复率摘要

### 涨停结构

口径:

- 首板
- 2板
- 3板
- 4板及以上

说明:

### 草原高手数据

口径:

- 数据来源于项目根目录下的 `草原高手数据` CSV 文件
- 文件名中的日期会被解析成快照日期
- 每份 CSV 会同步为一个高手持仓快照

更新方式:

- 手动放入新的 CSV 文件
- 运行 `pnpm run sync:experts`
- 前端和 MCP 都读取 `data/expert_holding_snapshots.json`

## 文件读取建议

- `研报 PDF/图片/Office`: 如果接入的 AI 支持原文件读取，优先直接读取文件本身，不必先转全文文本
- `文本型研报`: 可以直接读取文件，也可以通过 `get_research_report_content` 读取文本内容
- `高手 CSV`: 优先使用 MCP 返回的结构化数据，不建议让 AI 重新解析原始 CSV

- 当前 MCP 总览里已经包含结构摘要
- 后续可单独拆成独立 tool

### 题材持续性

口径:

- 取概念板块或行业板块最近 5 个交易日的历史排名
- 统计谁连续领涨
- 统计谁反复进入 Top3
- 统计同一题材强度变化

### 周期总览

口径:

- 结合龙头状态
- 涨停结构
- 修复率
- 量能趋势
- 高位风险

说明:

- 这是规则层摘要，不是纯主观解释
- 适合 AI 先读结论，再下钻明细

### 量能趋势

口径:

- 合并上证指数与深证成指日线成交额
- 核心判断趋势，不看固定金额

量能状态:

- `持续放量`
- `存量震荡`
- `缩量再缩量`
- `放量滞涨`

### 高位风险

口径:

- 统计 4板及以上样本
- 统计次日收盘跌幅小于等于 -8% 的高位 A 杀样本
- 统计高位次日收绿数量
- 结合炸板率判断风险等级

## AI 推荐分析顺序

建议 AI 按这个顺序读数据:

1. 先调用 `get_market_dashboard`
2. 再调用 `get_cycle_overview`
3. 再调用 `get_leader_state`
4. 再按需调用 `get_sentiment_snapshot`
5. 再调用 `get_sector_persistence`
6. 再按需调用 `get_volume_trend`
7. 再按需调用 `get_high_risk_panel`
8. 最后按需调用 `get_news_feed`
9. 如果需要读本地研报，先调用 `get_research_reports`
10. 再调用 `get_research_report_content`

原因:

- 先看总览，判断大方向
- 再看规则层阶段判断
- 再看龙头，确认高度和分歧
- 再看题材，确认主线是否持续
- 再看量能和高位风险，确认周期是否稳固
- 最后用新闻做解释，而不是反过来
- 如果需要引入基本面/行业资料，再读取本地研报

## AI 推荐输出结构

建议 AI 输出时固定使用这 5 段:

1. `市场阶段判断`
2. `量能与流动性状态`
3. `龙头与高标状态`
4. `主线题材与持续性`
5. `修复率与风险点`
6. `操作建议`

## 不推荐的做法

AI 不要这样做:

- 不要解析页面 HTML
- 不要依赖截图识别数字
- 不要把新闻当成比盘面更高优先级的信号
- 不要只看单一指标就直接下结论

## 后续建议

后续最值得补充到 MCP 的 tools:

1. `get_repair_rates`
   单独返回修复率历史

2. `get_limit_up_structure`
   单独返回涨停结构历史

3. `analyze_market_cycle`

## 结论

后续 AI 接入时，优先读取这份文档，再通过 MCP tools 取结构化数据。
不要让 AI 去“读页面”，而要让 AI “读数据”。
