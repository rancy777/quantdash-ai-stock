# 服务加固说明

更新时间：2026-04-17

## 实现思路

本轮加固目标不是重写业务，而是在不破坏现有页面行为的前提下，把高频接口、鉴权链路、异常处理和敏感配置补齐到可上线的最低安全线。

核心策略：

1. 高频接口统一走共享缓存。
   - 优先使用 Redis。
   - 未配置 Redis 时自动降级到进程内 TTL 缓存。
2. 外部 HTTP 访问统一走共享连接池。
   - 模型代理使用 `runtime.CLIENT`。
   - EastMoney 使用 `runtime.EASTMONEY_CLIENT`，显式 `trust_env=False`。
3. SQLite 访问统一收口到事务上下文。
   - 开启 `WAL`
   - 增加 `busy_timeout`
   - 自动 `commit/rollback`
4. 所有受保护的管理接口补权限校验。
5. 所有异常通过全局异常处理器统一输出，避免直接暴露堆栈。
6. 敏感配置读取时脱敏，保存/测试时自动合并已存密钥，避免“读出来是星号，保存后把真实密钥覆盖掉”。

## 接口文档

### 1. `GET /health`

用途：基础健康检查。

入参：无。

出参：

```json
{
  "status": "ok",
  "cache": {
    "backend": "memory",
    "redisEnabled": false
  }
}
```

错误码：

- `500` 服务内部错误

### 2. `POST /auth/register`

用途：注册账号。

入参：

```json
{
  "username": "demo_user",
  "password": "StrongPass123"
}
```

约束：

- `username` 仅允许 3-32 位字母、数字、点、下划线、中划线
- `password` 至少 8 位，必须包含大小写和数字

出参：

```json
{
  "token": "session-token",
  "username": "demo_user"
}
```

错误码：

- `400` 参数非法 / 用户名已存在
- `422` 请求体校验失败

### 3. `POST /auth/login`

用途：登录。

入参同注册。

出参同注册。

错误码：

- `400` 用户名或密码错误
- `429` 登录失败次数过多，触发限流
- `422` 请求体校验失败

### 4. `POST /auth/change-password`

用途：修改密码。

权限：必须登录。

请求头：

- `Authorization: Bearer <token>`

入参：

```json
{
  "oldPassword": "OldPass123",
  "newPassword": "NewPass456"
}
```

错误码：

- `400` 原密码错误 / 新密码过弱 / 新旧密码相同
- `401` 未登录或 token 失效

### 5. `GET /watchlist`

用途：获取远程自选。

权限：必须登录。

查询参数：

- `includeSignals: boolean`

错误码：

- `401` 未登录
- `500` 服务内部错误

### 6. `PUT /watchlist`

用途：保存远程自选。

权限：必须登录。

入参：`WatchlistEntry[]`

约束：

- 最多 500 条

错误码：

- `400` 数据非法
- `401` 未登录
- `422` 请求体校验失败

### 7. `GET /sync/runtime-status`

权限：必须登录。

出参：

## 统一错误响应

所有异常都通过统一异常处理器输出，不再直接返回 Python 堆栈。

统一格式：

```json
{
  "detail": "错误说明",
  "requestId": "req-xxxx"
}
```

参数校验失败格式：

```json
{
  "detail": "请求参数校验失败",
  "errors": [
    {
      "field": "body.username",
      "message": "String should have at least 3 characters"
    }
  ],
  "requestId": "req-xxxx"
}
```

## 部署注意事项

### Redis 缓存

可选环境变量：

- `QUANTDASH_REDIS_URL`
- `QUANTDASH_CACHE_PREFIX`

未配置时自动回退到内存缓存，但多进程部署时建议接 Redis。

### HTTP 连接池

环境变量：

- `HTTP_MAX_CONNECTIONS`
- `HTTP_MAX_KEEPALIVE_CONNECTIONS`
- `MODEL_PROXY_TIMEOUT_SECONDS`
- `MODEL_PROXY_CONNECT_TIMEOUT_SECONDS`

建议在生产环境明确设置，避免默认值过小或过大。

### IO 线程池

环境变量：

- `QUANTDASH_IO_WORKERS`

用于承接 SQLite 和文件 IO 的阻塞操作。

### 日志

当前已开启：

- 请求访问日志
- 统一错误日志
- 请求 ID 透传

生产环境建议由外部日志采集系统接管标准输出。
