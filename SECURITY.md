# Security Policy

## 报告方式

如果你发现下列问题，请不要公开提 issue：

- 密钥、令牌、Cookie、数据库或会话数据泄露
- 登录鉴权绕过
- 任意文件读取 / 写入
- 可导致远程命令执行的脚本或接口问题
- 第三方服务凭证误提交

优先使用 GitHub 的私密安全通道报告；如果仓库尚未启用相关能力，请先通过维护者的私下联系方式沟通，再决定是否公开披露。

## 维护者处理建议

收到安全问题后，建议按下面顺序处理：

1. 立即确认影响范围
2. 先撤掉或轮换泄露的密钥、令牌和第三方凭证
3. 清理公开历史中的敏感文件
4. 修复根因并补最小回归验证
5. 评估是否需要补充公开说明

## 仓库发布前重点检查

- `.env.local` 未被提交
- `data/auth.db`、`data/system/auth.db` 等本地数据库未被提交
- 自动下载的第三方研报原文、网页快照、临时调试文件未被提交
- README 和示例配置中不包含真实账号、真实 token、真实回调地址

## 支持范围

当前默认只保证仓库最新公开版本的安全修复处理。
