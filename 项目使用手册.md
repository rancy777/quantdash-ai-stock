# QuantDash 项目使用手册

本手册用于说明 `QuantDash A股看盘与复盘面板` 的页面功能、启动方式、数据同步链路、AI 集成、MCP 能力、后端接口、目录结构和常见操作流程。

适用范围：

- 前端页面使用
- 本地开发与启动
- 数据同步与日常维护
- 研报/新闻/高手数据使用
- AI 复盘、盘前计划、个股观察
- MCP / 飞书 / 本地模型接入

写在前面：

- 本项目默认遵循“`采集优先 Python，展示优先前端，Node 只做编排`”。
- 涉及“今天 / 明天 / 今日 / 次日”的地方，项目统一按“`最近交易日`”处理，不按自然日处理。
- 页面、MCP、飞书机器人统一复用同一份结构化数据，相关规则见 [STRUCTURED_DATA_RULES.md](./STRUCTURED_DATA_RULES.md)。

<a id="overview"></a>
## 1. 项目总览

### 1.1 项目定位

`QuantDash` 是一个基于 `Vite + React + TypeScript + FastAPI` 的 A 股分析面板，核心用途包括：

- 看盘：股票信息、K 线、情绪、板块、连板、龙头、风险面板
- 复盘：AI 当日复盘、超短深度分析、次日校验
- 盘前准备：盘前计划、观察标的、重点关注
- 资料整合：研报、本地上传资料、大 V 复盘、新闻聚合、高手持仓
- AI 接入：本地模型、云端模型、MCP、飞书机器人

### 1.2 技术架构

项目主要由 4 层组成：

1. 前端展示层：`App.tsx` 和 `components/*`
2. 前端服务层：`services/*`
3. 编排/同步层：`scripts/*.js`
4. Python 采集与后端层：`scripts/*.py`、`scripts/screener_service.py`

默认职责分工：

- Python：离线采集、全市场扫描、历史沉淀、后端 API
- 前端：展示、筛选、上传、交互、本地缓存
- Node：同步编排、启动脚本、MCP Server、兼容入口

### 1.3 左侧页面导航总表

前端主导航一共 9 个页面：

1. `股票信息`
2. `选股工具`
3. `连板天梯`
4. `板块周期`
5. `信息采集`
6. `情绪周期`
7. `AI对接`
8. `Skills`
9. `GitHub`

除此之外，左侧底部还有：

- `Data Status` 同步状态卡
- `Pro 版本` 升级提示卡

## 2. 环境准备与启动方式

### 2.1 运行前置

运行环境：

- `Node.js`
- `npm`
- `Python 3`
- `Chrome`

如果要完整使用以下功能，还需要补充：

- `研报同步`、`离线数据同步`、`后端服务`：安装 Python 依赖
- `NewsFilter 外网新闻抓取`：必须安装 Chrome，并能启动远程调试端口
- `Tushare`：准备 API Key
- `飞书机器人`：准备飞书应用凭据

### 2.2 安装依赖

前端依赖：

```bash
npm install
```

Python 依赖：

```bash
pip install -r requirements.txt
```

`requirements.txt` 主要包含：

- `fastapi`
- `uvicorn`
- `httpx`
- `openai`
- `playwright`
- `beautifulsoup4`
- `lxml`
- `requests`
- 翻译依赖

### 2.4 最快启动方式

只看前端页面：

```bash
npm install
npm run dev
```

完整启动方式：

```bash
npm run start:project
```

或：

```powershell
powershell -ExecutionPolicy Bypass -File .\start_project.ps1
```

macOS / Linux 也可以直接执行：

```bash
bash ./start_project.sh
```

### 2.5 `start_project.ps1` / `start_project.sh` 会做什么

[start_project.ps1](./start_project.ps1) 与 [start_project.sh](./start_project.sh) 会依次执行：

1. 切到项目根目录
2. 激活 `venv`
3. 输出 Python / Node / npm 版本
4. 如无 `node_modules` 则自动 `npm install`
5. 先调用 [stop_project.bat](./stop_project.bat) 清理旧进程
6. 启动后端：`python scripts\screener_service.py`
7. 启动前端：`npm run dev`
8. 启动 Chrome 远程调试实例：[start_newsfilter_chrome.bat](./start_newsfilter_chrome.bat)
9. 启动新闻采集器：`dingding盘中资讯2/dingding盘中资讯/run_both.py`

默认地址：

- 前端：`http://localhost:5173`
- 后端：`http://127.0.0.1:7878`
- NewsFilter Chrome 调试端口：`http://127.0.0.1:19223/json/version`

### 2.6 停止项目

执行：

```bash
npm run stop:project
```

或：

```bash
bash ./stop_project.sh
```

脚本会清理：

- `5173` 端口：前端
- `7878` 端口：后端

注意：

- 新闻采集器和 Chrome 调试实例是单独窗口启动的，如需彻底关闭，可能还需要手动关掉对应命令行和 Chrome 进程。

## 3. 页面功能手册

<a id="ui-stock"></a>
### 3.1 股票信息

页面职责：

- 全局股票搜索
- 股票列表查看
- 选中股票详情
- 悬浮 K 线大卡片查看
- 接收来自 AI 页的“跳转到个股”请求

主要交互：

1. 顶部搜索框支持按 `股票代码` 或 `股票名称` 搜索
2. 点击表格行切换主展示股票
3. 鼠标悬停表格行约 `400ms` 后打开大号悬浮卡
4. AI 页触发“去股票信息页”时，会自动跳到本页并定位对应股票

页面显示内容：

- 当前价格
- 涨跌幅
- 成交量
- 成交额
- 市盈率
- 总市值
- 行业
- 概念标签
- 东方财富来源实时行情表

悬浮卡支持内容：

- 主图 K 线
- `MA` 均线开关
- `BOLL`
- 副图指标：`MACD / KDJ / RSI / BIAS / WR / VR`
- 缠论结构开关

缠论结构包含：

- 包含关系处理
- 分型
- 笔
- 线段
- 中枢

补充：

- 用于辅助观察，不是严格交易信号引擎
- 还没有接入背驰、一二三类买卖点、多周期联立

数据来源：

- `services/quotesService.ts`
- `services/chanService.ts`
- `components/StockHoverCard.tsx`

<a id="ui-screener"></a>
### 3.2 选股工具

页面职责：

- 运行策略选股
- 浏览候选结果
- 悬浮查看候选股 K 线细节

内置策略：

1. `创业板2连板回调3天`
2. `涨停回调低吸`
3. `涨停回调五日线N字`
4. `涨停回调不破低点`

使用步骤：

1. 左侧选择策略
2. 点击 `开始筛选`
3. 等待系统获取目标池并进行形态匹配
4. 在右侧查看结果卡片
5. 鼠标悬停结果卡片可打开大悬浮分析卡

页面展示内容：

- 策略说明
- 扫描进度条
- 扫描状态文本
- 命中标的数量
- 概念涨停统计
- 命中标的卡片列表

补充：

- `选股工具` 前端会走 `quotesService.ts` 的候选池 + 形态判断
- Python 后端 `scripts/screener_service.py` 也支持策略扫描和问财查询
- 若真实结果为空，页面和后端都可能注入演示样本，方便验证 UI

相关服务：

- `services/quotesService.ts`
- `scripts/screener_service.py`

<a id="ui-ladder"></a>
### 3.3 连板天梯

页面职责：

- 查看近 20 个交易日连板梯队
- 单日复盘
- 昨今对比
- 连板个股追踪

核心功能：

1. `多日模式`
   查看近 20 个交易日的连板矩阵表
2. `单日模式`
   查看某一天每个连板层级下的个股
3. `今昨对比`
   在单日模式下，对比昨日各层级与今日晋级/掉队情况
4. `连板追踪`
   开启彩色链路，高亮同名个股在不同日期的连续路径
5. `概念涨停统计`
   单日模式下统计当日涨停概念分布
6. `日期切换`
   支持前后翻天和手工选日期
7. `悬浮卡`
   对任意连板股悬浮查看大卡片

页面重点数据：

- 连板层级
- 个股名称与代码
- 涨跌幅
- 最后涨停时间
- 概念/行业标签
- 昨日晋级、今日停留、未晋级、收盘涨跌幅

数据来源：

- `services/stockService.ts`
- `services/limitUpPoolService.ts`
- `services/quotesService.ts`

<a id="ui-sector"></a>
### 3.4 板块周期

页面职责：

- 看板块轮动
- 看题材持续性
- 切换概念板块与行业板块

核心功能：

1. `概念 / 行业` 切换
2. 最近 5 个交易日热力轮动表
3. 当前最强题材
4. 连续领涨天数
5. 近 5 日 Top3 出现次数
6. 题材持续性轨迹卡片

页面主要指标：

- `currentLeaderName`
- `currentStreakDays`
- `currentTopThreeAppearances`
- `strongestRepeatName`
- `strengthDelta`

查看顺序：

- 先看当前领涨板块
- 再看是否连续领涨、是否反复进 Top3
- 最后结合情绪周期判断它是主线、轮动还是脉冲

数据来源：

- `services/sectorService.ts`

## 4. 数据同步、采集与批处理脚本

### 4.1 同步总原则

项目同步规则：

- 批量采集、历史沉淀、全市场扫描优先 Python
- 前端优先读取本地文件，缺文件时再兜底接口
- Node 负责串联与编排

### 4.2 常用 npm 脚本总表

| 命令 | 作用 |
| --- | --- |
| `npm run dev` | 启动前端开发环境 |
| `npm run build` | 构建前端 |
| `npm run preview` | 预览构建产物 |
| `npm run fetch:data` | 主市场数据同步 |
| `npm run sync:all` | 全量同步编排入口 |
| `npm run sync:market:py` | Python 主市场数据同步 |
| `npm run sync:kline:py` | Python K 线库同步 |
| `npm run sync:cycle` / `sync:cycle:py` | 情绪周期快照同步 |
| `npm run sync:stocks` / `sync:stocks:py` | 股票快照同步 |
| `npm run sync:sectors` / `sync:sectors:py` | 板块快照同步 |
| `npm run sync:offline:py` | Python 完整离线同步 |
| `npm run fetch:data:auto` | 自动盘后同步 |
| `npm run sync:startup-check` | 启动前快照检查与修复 |
| `npm run sync:emotion` | 跨市场情绪指标同步 |
| `npm run sync:reports` | 研报同步 |
| `npm run sync:experts` | 高手持仓快照同步 |
| `npm run mcp:server` | 启动本地 MCP Server |
| `npm run feishu:bot` | 启动飞书机器人 |

### 4.3 数据批处理入口

[获取单天数据.bat](./获取单天数据.bat) 用于导出指定交易日的完整项目快照。

[补全之前数据.bat](./补全之前数据.bat) 用于手工补全历史数据。

`补全之前数据.bat` 菜单项：

1. `All previous data`：完整 Python 离线同步，走 `sync:offline:py`
2. `Market core`：主市场快照，走 `sync:market:py`
3. `Stock snapshots`：股票快照，走 `sync:stocks:py`
4. `Sector snapshots`：板块快照，走 `sync:sectors:py`
5. `Sentiment cycle`：情绪周期，走 `sync:cycle:py`
6. `Emotion indicators`：跨市场情绪指标，走 `sync:emotion`
7. `Kline library`：K 线库，走 `sync:kline:py`

选择 `Sector snapshots` 时，可继续选：

- `All`：概念板块 + 行业板块
- `Concept`：仅概念板块
- `Industry`：仅行业板块

说明文件见 [获取数据命令.txt](./获取数据命令.txt)。

### 4.4 启动前同步检查

默认由 `start_project.ps1` 触发。

可用模式：

- `startup`
- `market`
- `offline`

控制变量：

- `STARTUP_AUTO_SYNC=0`：跳过启动时自动同步
- `STARTUP_SYNC_MODE=startup`：轻量模式
- `STARTUP_SYNC_MODE=market`：主市场同步
- `STARTUP_SYNC_MODE=offline`：完整离线同步

### 4.5 主市场数据同步

`fetch:data` 会更新：

- `sentiment.json`
- `performance.json`
- `ladder.json`
- `stock_list_full.json`
- `stock_list_chinext.json`
- `sector_rotation_*.json`
- `sector_persistence_*.json`
- `limit_up_structure.json`
- `repair_rate.json`
- `leader_state.json`
- `market_volume_trend.json`
- `high_risk.json`
- `cycle_overview.json`

### 4.6 K 线库同步

相关文件：

- `scripts/fetch_kline_library.py`
- `scripts/syncKlineLibraryPy.js`
- `data/klines/*`
- `data/kline-manifest.json`

策略说明：

- 默认只补最近窗口

- 可用 `KLINE_RECENT_LIMIT` 调整窗口
- 可用 `KLINE_FORCE_FULL=1` 强制全量补齐

### 4.7 研报同步

研报同步走 Python 采集链路：

- 抓东方财富个股研报

- 保留东财聚合页和天风证券页面快照

- `scripts/fetch_research_reports.py`

- `scripts/researchReportSources.js`

常用变量：

- `REPORT_INCLUDE_SNAPSHOTS`
- `REPORT_ITEM_LIMIT_OVERRIDE`
- `REPORT_DOWNLOAD_PDFS`

### 4.8 高手持仓同步

相关链路：

- 手动把 CSV 放到 `草原高手数据/`
- 执行 `npm run sync:experts`
- 生成 `data/expert_holding_snapshots.json`
- 前端和 MCP 统一读取结构化结果

### 4.9 新闻采集

新闻链路由两个部分组成：

1. `财联社` 电报流
2. `NewsFilter / 外网新闻` 聚合流

关键目录：

- `dingding盘中资讯2/dingding盘中资讯/`

关键文件：

- `run_both.py`
- `cls_telegraph_to_dingtalk_single.py`
- `newsfilter_to_dingtalk.py`
- `news_store.py`

Chrome 调试入口：

- [start_newsfilter_chrome.bat](./start_newsfilter_chrome.bat)

补充：

- 如果 Chrome 没装好或远程调试端口没起来，`CLS` 链路仍可跑，`NewsFilter` 外网新闻链路可能失败

### 4.10 同步状态文件

以下命令会写入同步状态：

- `fetch:data`

## 8. 常见操作流程

### 8.1 首次安装后跑起来

1. 安装 Node 和 Python
2. 执行 `npm install`
3. 执行 `pip install -r requirements.txt`
4. 按需填写 `.env.local`
5. 双击 `start_project.bat`

### 8.2 只开前端看页面

1. `npm install`
2. `npm run dev`
3. 打开 `http://localhost:5173`

补充：

- 这种方式下，若后端、新闻采集器、同步状态接口没有启动，部分功能会降级或缺失。

### 8.3 盘后数据更新

操作顺序：

1. 双击 `补全之前数据.bat`
2. 选 `1` 运行完整离线同步，或按需要选单个数据模块
3. 刷新页面查看 `Data Status`
4. 进入 `情绪周期`、`板块周期`、`连板天梯` 校验数据

### 8.4 同步研报

1. 执行 `npm run sync:reports`
2. 打开 `信息采集 -> 研报`
3. 按来源、日期、股票代码或机构筛选
4. 需要时点击 `AI 摘要`

### 8.5 导入个人资料或大 V 复盘

1. 打开 `信息采集 -> 大V复盘`
2. 新建条目或选中已有条目
3. 填写标题、作者、来源、标签
4. 粘贴正文或导入文本文件
5. 内容自动保存在浏览器本地

### 8.6 生成 AI 当日复盘

1. 打开 `AI对接 -> 模型接入`
2. 配好模型 `base url / key / model`
3. 切到 `AI 生成`
4. 选择模型
5. 点击生成 `AI 当日复盘`
6. 生成后可复制、编辑、下载 MD

### 8.7 生成盘前计划并加入重点关注

1. 先有一份最近交易日 `AI 当日复盘`
2. 在 `AI 生成` 点击生成 `盘前计划`
3. 系统自动抽取观察标的代码
4. 一键加入 `自选 / 重点关注`
5. 已登录则写后端，自否则写浏览器本地

### 8.8 做个股观察

1. 打开 `AI 生成 -> 个股观察`
2. 输入股票代码
3. 生成个股观察
4. 可查看相关资讯、相关研报、已有 AI 研报摘要
5. 需要图表时可点 `去股票信息页`

### 8.9 飞书问答接入

1. 在 `.env.local` 或 `AI对接 -> 飞书` 配置飞书信息
2. 执行 `npm run feishu:bot`
3. 飞书开放平台改为长连接接收事件
4. 在飞书里发送文本提问

### 8.10 导出某一天的完整项目快照

方式一：

1. 双击 `获取单天数据.bat`
2. 输入 `YYYY-MM-DD`

方式二：

```bash
node scripts/exportSingleDayProjectData.js YYYY-MM-DD
```

输出位置：

- `data/single_day_snapshots/YYYY-MM-DD.json`

## 9. 常见问题与注意事项

### 9.1 为什么页面有时读本地，有时走接口

这是项目的读取策略：

- 先读本地结构化文件
- 缺文件时再接口兜底

目的：

- 页面速度
- 历史回放
- AI / MCP 复用

### 9.2 为什么“今天”不一定是自然日

AI 复盘、盘前计划、情绪快照很多地方按 `最近交易日` 工作，不是系统日期的今天。

### 9.3 NewsFilter 外网新闻为什么抓不到

先检查：

1. Chrome 是否安装
2. `start_newsfilter_chrome.bat` 是否成功启动
3. `19223` 调试端口是否可访问
4. `dingding盘中资讯2` 的采集脚本是否正常运行

### 9.4 为什么“立即同步”没有效果

检查项：

1. 后端 `scripts/screener_service.py` 是否已启动
2. `7878` 端口是否可访问
3. 前端是否能读取 `sync_status.json`

### 9.5 为什么 AI 生成失败

常见原因：

- 模型线路没填完整
- `base url` 不对
- `api key` 无效
- 模型协议选错
- 后端模型代理超时

### 9.6 为什么加入重点关注后没看到

项目有两套模式：

- 已登录：保存到后端 watchlist
- 未登录：保存到浏览器本地 focus list

先确认当前使用的是哪种模式。

### 9.7 为什么有些数据目录同时存在新旧两套

这是目录迁移策略：

- 新路径优先
- 旧路径兜底

不是重复错误，而是兼容过渡。

### 9.8 为什么部分策略会出现演示股

当实时扫描结果为空时，前端或后端会注入演示样本，用于验证页面流程和交互。
