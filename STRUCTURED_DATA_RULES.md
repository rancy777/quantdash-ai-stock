# QuantDash Structured Data Rules

这份文档定义本项目后续新增“界面显示数据”功能时必须遵守的规则。

核心原则只有一条：

所有新增到界面的数据，必须先形成结构化数据对象，再由页面消费。

不要让页面直接承担数据采集、页面文本拼接、DOM 反解析、截图识别、临时字符串拆分等职责。

## 目标

- 保证页面和 AI 读取同源数据，而不是两个口径
- 保证 MCP、前端页面、飞书机器人、后续自动化流程都能复用同一份数据
- 保证新增功能可以沉淀、缓存、回放、调试和校验
- 避免“页面能看，AI 不能稳定读”“今天能跑，后面无法维护”的实现

## 硬规则

以后只要新增一种会显示在界面的数据，必须满足下面 6 条：

1. 必须先定义结构化数据对象
2. 必须明确数据来源
3. 必须明确服务层入口
4. 必须明确本地落盘或缓存策略
5. 必须让页面只消费对象，不直接拼接原始源数据
6. 如果该数据未来可能被 AI 使用，必须保证 MCP 或服务层可直接读取这份结构化结果

## 推荐数据流

统一采用下面的数据流：

`外部数据源 / Python脚本 / 后端接口 / 本地文件`
-> `结构化清洗`
-> `data/*.json 或服务层对象`
-> `services/*`
-> `components/*`
-> `AI / MCP / 其他集成复用`

不要采用下面的数据流：

`外部网页`
-> `页面临时 fetch`
-> `组件内部随手 reshape`
-> `直接渲染`
-> `AI 再反过来读页面`

## 结构化对象要求

新增数据时，至少要满足这些要求：

- 字段名稳定，可复用，不要今天叫 `value` 明天叫 `num`
- 字段语义清晰，不要把多个意思塞进一个字符串
- 日期字段尽量标准化，优先使用 `YYYY-MM-DD` 或 ISO 时间
- 数值字段保持数值类型，不要提前格式化成带单位字符串
- 枚举字段要可预期，例如 `sourceType`、`riskLevel`、`stage`
- 保留必要元信息，例如 `source`、`updatedAt`、`tradeDate`、`symbol`

## 页面层规则

页面层只做这些事情：

- 调用服务层
- 读取结构化对象
- 排序、筛选、展示
- 少量纯展示格式化

页面层不要做这些事情：

- 直接请求复杂外部数据源并自己拼业务口径
- 在组件里解析 HTML、CSV、大段文本
- 在组件里推导核心业务结论
- 让不同页面各自维护一套同名但不同结构的数据

## 服务层规则

所有界面数据都应该有服务层入口。

推荐做法：

- 本地文件读取走 `services/localDataService.ts`
- 东方财富等外部接口读取走独立 `services/*`
- 批量采集、盘后沉淀、历史归档优先放到 `scripts/*.py`
- 页面不要绕过服务层直接依赖脚本产物细节

如果新增一种数据，至少应该能回答这 4 个问题：

- 这个数据从哪里来
- 这个数据最终长什么结构
- 页面通过哪个 service 读取
- AI 或 MCP 以后如何复用

## AI 相关规则

本项目的 AI 功能不应该读页面 DOM。

以后新增任何可能被 AI 复盘、计划、摘要、观察功能使用的数据，都应遵守：

- AI 读取结构化对象，不读取页面渲染结果
- 如果页面显示了关键数据，AI 最好能读到同一份对象
- 不要让 AI 依赖截图、手工复制文本、页面卡片文案
- 能进入 MCP 的数据，优先补成 MCP tool 或复用同口径服务层结果

## 新增功能时的最小交付清单

以后每新增一种界面数据，提交前至少检查下面这些项：

- 是否定义了明确的 TypeScript 类型或 Python 输出结构
- 是否有稳定字段，而不是临时对象
- 是否经过服务层统一读取
- 是否有本地文件或缓存落点
- 是否避免在组件里直接做重业务逻辑
- 是否考虑以后给 AI / MCP 复用
- 是否能区分“原始数据”“结构化结果”“展示文案”

## 推荐示例

正确做法：

- Python 脚本抓取行情或研报
- 清洗后写入 `data/*.json`
- `services/*` 读取并补少量衍生字段
- 页面直接展示
- AI 通过同一份服务层数据或 MCP 数据分析

错误做法：

- 组件里直接 `fetch` 外部网页
- 组件里用正则硬拆页面文本
- 页面显示靠一套临时字段
- AI 再去读页面文本或截图

## 对以后开发的要求

从这份文档开始，默认把“结构化数据先行”视为项目约束，不再把它当成可选优化项。

以后如果要新增任何“获取别的数据并显示到界面”的功能，先做数据结构设计，再做服务层，再做页面。

顺序不要反过来。
