# 第二数据源说明

本文档描述 QuantDash 当前的第二数据源能力、启用方式、支持范围和排障方式。

## 1. 当前目标

第二数据源不是用来替代所有 EastMoney 数据，而是作为：

- `K 线 / 基础行情` 的备用来源
- `市场宽度 / 全市场快照 / 指数序列` 的备用来源
- EastMoney 网络不稳定时的自动回退来源
- 本机环境排障和对比验证的辅助来源

默认策略仍然是 `主源优先`，所以即使代码已经接入第二源，只要你没有切换策略，也不会影响现在的主链路。

## 2. 当前第二源

- Provider: `mootdx`
- 启用前提：本机已安装 `mootdx`

建议安装方式：

```powershell
pip install -U "mootdx[all]"
```

如果你只装核心包，也可以先尝试：

```powershell
pip install -U mootdx
```

## 3. 当前支持范围

### 3.1 已支持第二源的数据集

- 个股 `K 线`
- 指数 `K 线`
- 主要指数快照
- 股票列表 / 基础行情
- 市场宽度概览
- 全市场原始快照
- 全市场涨跌分布快照
- 指数情绪序列
- 指数成交额序列

### 3.2 明确只走主源的数据集

以下数据集目前保留为 `EastMoney-only`：

- 涨停池 / 炸板池 / 跌停池
- 板块榜单 / 板块历史
- A 股平均估值
- 股指期货主力与净持仓

原因很简单：这些数据并不是 `mootdx` 的直接优势区，强行接第二源只会制造“看起来支持，实际字段不等价”的问题。

## 4. 策略模式

当前支持三档策略：

### 4.1 主源优先

- 默认只走 EastMoney
- 第二源只做状态展示
- 适合日常使用

### 4.2 失败自动切第二源

- 先走 EastMoney
- 如果主源超时、失败或被冷却，再自动切到 `mootdx`
- 适合网络偶发不稳定时使用

### 4.3 优先第二源

- 支持的数据集直接优先走 `mootdx`
- EastMoney 作为兜底
- 适合临时避障或对比测试

## 5. 按数据集覆盖

策略除了全局模式，还支持按数据集覆盖：

- `stock_kline`
- `index_kline`
- `major_indexes`
- `stock_quote_list`
- `market_breadth`
- `index_series`

这意味着你可以：

- 让 `K 线` 优先走 `mootdx`
- 但 `主要指数快照` 仍然走 EastMoney
- 或者只让 `股票列表` 在主源失败时自动切第二源

## 6. UI 位置

当前设置入口在左侧 sidebar：

- `Data Status`
- `第二数据源策略`

该卡片会显示：

- `mootdx` 是否已安装
- 当前全局模式
- 支持的数据集数量
- 最近探测时间
- 最近成功时间
- 最近错误
- 最近测速耗时
- 各数据集探测结果

高级设置默认折叠，避免把 sidebar 变成控制台。

## 7. 页面来源徽标

高频页面的数据徽标现在会更细地展示：

- `实时采集 · EastMoney`
- `代理缓存 · EastMoney`
- `最近快照 · EastMoney`
- `第二数据源 · Mootdx`
- `本地文件 · 本地`
- `模拟回退 · 模拟`

如果是快照回退或缓存命中，还会额外显示：

- `快照回退`
- `缓存命中`

## 8. 健康检查与测速

后端已支持第二源健康状态汇总：

- `GET /eastmoney/status`
- `GET /eastmoney/secondary-health`
- `POST /eastmoney/secondary-health/probe`

其中 `probe` 会执行轻量样本探测，覆盖：

- 个股 K 线
- 指数 K 线
- 主要指数快照
- 股票列表 / 基础行情
- 市场宽度
- 指数情绪 / 成交额序列

返回信息包括：

- 最近探测时间
- 最近成功时间
- 最近错误
- 平均耗时
- 各数据集的探测结果

## 9. 安装后 smoke test

安装 `mootdx` 后，可以直接运行：

```powershell
python .\scripts\smoke_mootdx_provider.py
```

它会输出：

- 当前 provider policy
- 当前第二源健康状态
- 如果已安装 `mootdx`，还会执行一次轻量 probe

## 10. 排障建议

### 10.1 UI 显示 `mootdx 未就绪`

说明当前 Python 环境里没有检测到 `mootdx`。

先确认：

```powershell
python -c "import mootdx; print(mootdx.__file__)"
```

### 10.2 已安装，但探测失败

优先检查：

- 是否装在项目实际运行的 Python 环境里
- 通达信行情链路是否可连
- 本机安全软件或代理是否干扰了连接

### 10.3 页面仍然显示 EastMoney

说明：

- 你当前策略仍是 `主源优先`
- 或该数据集本身属于 `EastMoney-only`
- 或 `mootdx` 探测不可用，系统自动保留主源

### 10.4 为什么不把所有数据都切到第二源

因为第二源的价值在于“补足 K 线 / 基础行情链路”，不是伪装成一个全能 A 股数据平台。

对于涨停池、板块榜单、A 股平均估值、股指期货这类数据，继续保持 EastMoney 或本地快照更稳。

## 11. 验收清单

### 11.1 未安装 `mootdx`

应满足：

- `GET /eastmoney/status` 中 `secondaryAvailable=false`
- sidebar 显示 `mootdx 未就绪`
- 页面来源徽标继续显示 `EastMoney / 本地 / 快照`
- `python .\scripts\smoke_mootdx_provider.py` 返回 `available=false`

### 11.2 已安装 `mootdx`

应满足：

- `GET /eastmoney/provider-policy` 可返回支持的数据集清单
- `POST /eastmoney/secondary-health/probe` 可返回各数据集探测结果
- sidebar 可保存全局模式和数据集覆盖模式
- `stock_kline / index_kline / major_indexes / stock_quote_list / market_breadth / index_series` 在策略命中时可切第二源

### 11.3 推荐校验命令

```powershell
python -m compileall scripts/server scripts/smoke_mootdx_provider.py
npm run typecheck
npm run lint
npm run build
python .\scripts\smoke_mootdx_provider.py
```

### 11.4 页面级检查点

- 股票信息页：来源徽标能区分 `EastMoney` 与 `Mootdx`
- 板块周期页：来源徽标仍正确显示本地/实时/快照
- 连板天梯页：来源徽标可显示快照回退
- sidebar：第二源策略卡能展示最新健康状态与测速结果
